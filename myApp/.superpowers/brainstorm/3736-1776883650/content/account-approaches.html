A — Single scrollable page

All sections stacked top-to-bottom: user info → liked pros grid → become a pro / mode toggle → logout. Simple, fast to build, no new routes.

B — Settings-style list (iOS)

Profile card header + grouped rows like iOS Settings. Liked Pros row navigates to a separate screen. Clean hierarchy, familiar pattern.

C — Profile-first with horizontal scroll

Centered profile avatar at top, liked pros shown as a horizontal scroll row (like Instagram highlights). More visual but slightly complex.

My recommendation: B (Settings-style) — familiar iOS pattern, cleanest separation between liked pros and settings rows, easy to extend later.
